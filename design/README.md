# CedarEx 工具台交互设计图

## 设计目标

- 按非代码用户使用习惯优化为 CedarEx 工具台。
- 保留必要界面结构：工具列表、中心工作区、底部执行过程、状态栏。
- 顶部标题栏只保留窗口控制点，删除命令搜索和环境信息，减少视觉噪音。
- 底部区域不再强调终端概念，只保留单一执行过程、状态提示和记录操作。
- 界面文案尽量使用中文，减少脚本、参数、终端、Output、AI Review 等偏技术表达对非代码用户的干扰。
- 软件默认固定为亮色主题，不跟随系统深色/浅色设置。
- 界面文字默认允许选中复制，便于复制工具名、说明、执行记录和错误信息。
- `使用说明` 标签页展示当前工具目录下真实 `README.md` 的解析结果。
- HTML 设计稿支持基础交互：切换标签、选择工具、填写信息、模拟运行、展开折叠和拖拽分隔条。
- 不再设置独立智能助手面板；AI 问答视为在执行过程中输入问题，回答也显示在执行过程文本流中。
- 主操作只放在底部执行过程区域，底部只保留 `运行` 和 `停止` 两个按钮。
- 运行完成后不在中心区域展示报告卡片，只在底部执行过程中显示状态和记录。
- 中心配置区不再展示工具标题、描述和准备状态，选择工具后直接展示表单。

## 设计图文件

- `vscode-script-workbench.html`：1440x900 交互 HTML 设计图。
- `workbench.css`：固定亮色主题、Workbench 布局、滚动条、分隔条和控件样式。
- `workbench.js`：工具树、标签切换、模拟运行、折叠展开和拖拽分隔条交互。
- `layout-sketch.txt`：文本版页面布局示意。

## 低耦合结构

- HTML 只描述 Workbench DOM 结构。
- CSS 只负责视觉主题、布局和响应式/滚动规则。
- JS 只负责状态、事件和示例数据。
- 设计稿不再把样式和交互混写在 HTML 里，便于后续迁移到 pywebview + 轻量 Workbench 实现。

## 布局说明

1. 顶部窗口栏：只保留窗口控制点。
2. 工具列表：显示搜索框、工具数量、刷新入口和常用工具列表；工具很多时左侧内部滚动。
3. 中心工作区：包含 `填写信息`、`使用说明` 两个标签；`使用说明` 解析真实 `README.md`；配置内容超过可用高度时在中心区内部滚动。
4. 执行过程：底部固定高度展示执行过程、状态提示、`运行` 和 `停止` 两个按钮；日志和 AI 问答都在这里显示。
5. 状态栏：展示 CedarEx 工具台状态信息。

## 滚动规则

- 整体窗口不滚动，参考 VSCode 的固定工作台布局。
- 左侧采用接近 VSCode 资源管理器的紧凑树，不使用卡片式工具项；工具多时左侧内部滚动。
- 左侧工具列表、中心配置区、底部执行过程分别拥有自己的滚动条。
- 左侧工具列表宽度、底部执行过程高度支持拖拽调整，参考 VSCode 分隔条交互。
- 中心配置/说明区域默认最多占中间工作区约 2/3 的高度，内容更多时只滚动中心区，不挤压底部执行过程。
- 底部执行过程保持固定高度，避免运行日志过多时影响表单操作。
